# Ugen Templates — документация для фронтенда (пользовательское приложение)

Эта дока описывает всё, что нужно фронтенду **пользовательской части** для работы с шаблонами:
создание шаблона пользователем, просмотр каталога, и **использование (покупка) шаблона** — создание
проекта из шаблона со списанием денег с баланса.

> ⚠️ Установка цены шаблона — **НЕ** в этой доке. Цену задаёт только админ-панель отдельным
> эндпоинтом. См. `ugen_template_pricing_admin.md`.

---

## 1. Суть фичи

- Шаблоны создают **сами пользователи** из своих проектов. Шаблон создаётся **бесплатным** (`price = 0`).
- **Пользователь НЕ может задать цену** ни при создании, ни при обновлении шаблона — поля цены в этих
  запросах просто нет.
- Цену шаблону назначает **администратор** через админ-панель.
- Когда пользователь нажимает «использовать шаблон» (создать проект из шаблона):
  - если шаблон **бесплатный** (`price = 0`) — проект создаётся как раньше, без биллинга;
  - если шаблон **платный** (`price > 0`) — **сначала списывается цена с баланса текущего проекта**
    пользователя, потом создаётся проект. Если денег не хватает — приходит `402`, проект не создаётся.
    Если создание проекта упало уже после списания — деньги **автоматически возвращаются**.

## 2. Общие правила API

### Формат ответа (обёртка)
Все ответы завёрнуты в единый конверт:
```json
{
  "status": "OK",
  "description": "The request has succeeded",
  "data": { ... },          // полезная нагрузка (объект/массив/строка)
  "custom_message": ""
}
```
- При ошибке `data` обычно содержит текст ошибки (строка), а `status`/HTTP-код указывают на причину.

### HTTP-коды, которые нужно обрабатывать
| Код | status | Когда |
|-----|--------|-------|
| 200 | `OK` | Успех |
| 400 | `BAD_REQUEST` / `INVALID_ARGUMENT` | Невалидное тело/параметры (например, кривой JSON, невалидный `id`) |
| 401 | `UNAUTHORIZED` | Нет/невалидный токен |
| 402 | `PAYMENT_REQUIRED` | **Недостаточно средств** на балансе проекта для платного шаблона |
| 404 | `NOT_FOUND` | Шаблон или проект не найдены |
| 500 | `GRPC_ERROR` | Внутренняя ошибка сервера |

### Авторизация
Все эндпоинты, кроме публичного каталога, требуют стандартные заголовки авторизации
(те же, что уже используются для остальных `/v1` запросов приложения):
```
Authorization: Bearer <token>
Environment-Id: <environment_id>
Resource-Id: <resource_id>
```
`project_id` / `environment_id` берутся сервером из токена/заголовков — отдельно в теле их слать не нужно.

## 3. Модель объекта шаблона (`UgenTemplate`)

Возвращается во всех ответах, где фигурирует шаблон.

```json
{
  "id": "9b1f...uuid",
  "name": "CRM Starter",
  "description": "Готовая CRM",
  "photo": "https://.../cover.png",
  "mcp_project_id": "uuid",
  "order_number": 10,
  "created_at": "2026-07-02T10:00:00Z",
  "updated_at": "2026-07-02T10:00:00Z",
  "deleted_at": 0,
  "preview_url": "https://preview.example.com",
  "source_resource_env_id": "uuid",
  "source_project_id": "uuid",
  "source_environment_id": "uuid",
  "source_node_type": "node-1",
  "source_mcp_resource_env_id": "uuid",
  "source_function_id": "uuid",
  "source_repo_id": "uuid",
  "images": ["https://.../1.png", "https://.../2.png"],
  "like_count": 12,
  "dislike_count": 1,
  "current_user_reaction": "like",
  "price": 50000,
  "currency_id": "88c816a3-24e8-4994-ab70-9bc826bb9dc3"
}
```

**Ключевые поля для UI цены:**
- `price` (number) — цена шаблона. **`0` = бесплатный** (не показывать бейдж цены / кнопка «Использовать бесплатно»).
- `currency_id` (string, uuid валюты) — валюта цены. Пусто (`""`) трактуется как UZS. Для отображения символа
  валюты сопоставляйте `currency_id` со справочником валют. Если `price = 0`, `currency_id` можно игнорировать.
- `current_user_reaction` — `"like"`, `"dislike"` или `""` (реакция текущего пользователя; заполняется только при наличии токена).

> **Как показывать цену:** если `price > 0` — «Платный шаблон, {price} {валюта}»; если `price === 0` — «Бесплатно».

## 4. Каталог шаблонов

### 4.1. Публичный список (без авторизации)
```
GET /v1/ugen-template/public?limit=10&offset=0
```
- `limit` (по умолчанию 10), `offset` (по умолчанию 0).
- `current_user_reaction` будет `""`, если запрос без токена.

**200:**
```json
{
  "status": "OK",
  "data": {
    "templates": [ { /* UgenTemplate */ } ],
    "count": 42
  }
}
```

### 4.2. Публичный один шаблон (без авторизации)
```
GET /v1/ugen-template/public/:id
```
**200:** `data` = объект `UgenTemplate`.
**400:** невалидный `id`. **404/500:** не найден / ошибка.

### 4.3. Список/один с авторизацией (для current_user_reaction)
Те же данные, но с токеном возвращается корректный `current_user_reaction`:
```
GET /v1/ugen-template?limit=10&offset=0
GET /v1/ugen-template/:id
```

## 5. Создание шаблона пользователем  ⚠️ ИЗМЕНИЛОСЬ

```
POST /v1/ugen-template
```

### ❗️ Что изменилось
Из тела запроса **убраны любые поля цены**. Пользователь создаёт шаблон **всегда бесплатным**
(`price = 0`). Даже если фронт пришлёт `price`/`currency_id` — они **игнорируются** сервером.
Цену потом назначает админ (см. админскую доку).

### Тело запроса
```json
{
  "name": "CRM Starter",
  "description": "Готовая CRM",
  "photo": "https://.../cover.png",
  "mcp_project_id": "uuid",             // обязательно
  "order_number": 10,
  "preview_url": "https://preview...",  // если пусто — подтянется из функции-источника
  "source_function_id": "uuid",         // обязательно
  "source_resource_env_id": "uuid",     // если пусто — резолвится из текущего проекта
  "source_mcp_resource_env_id": "uuid", // если пусто — резолвится из текущего проекта
  "source_repo_id": "uuid",             // если пусто — берётся из функции-источника
  "images": ["https://.../1.png"]
}
```
> `price` и `currency_id` в этом запросе **отсутствуют**. Не отправляйте их.

**200:** `data` = созданный `UgenTemplate` (с `price: 0`, `currency_id: ""`).
**400:** невалидное тело. **400 (INVALID_ARGUMENT):** не разрешились обязательные source-поля. **500:** ошибка.

---

## 6. Обновление шаблона пользователем  ⚠️ ИЗМЕНИЛОСЬ

```
PUT /v1/ugen-template/:id
```
Тело — как у создания (name, description, photo, source-поля, images, order_number).

### ❗️ Что изменилось
Обновление **тоже не трогает цену** — полей `price`/`currency_id` в теле нет, текущая цена шаблона
сохраняется без изменений. Менять цену можно только админским эндпоинтом.

**200:** `data` = обновлённый `UgenTemplate`.

---

## 7. Реакции (лайк/дизлайк) — без изменений
```
POST   /v1/ugen-template/:id/reaction     Body: { "reaction_type": "like" | "dislike" }
DELETE /v1/ugen-template/:id/reaction      // снять свою реакцию
GET    /v1/ugen-template/:id/reactions?reaction_type=like&limit=10&offset=0
```
`like_count` / `dislike_count` / `current_user_reaction` в объекте шаблона отражают агрегаты.

## 8. 🚀 Использование шаблона (создание проекта + оплата)

Главный флоу. Пользователь выбирает шаблон и создаёт из него свой проект. Если шаблон платный — с
баланса **текущего проекта** пользователя списывается цена.

```
POST /v1/ugen-template/create-project
```

### Тело запроса
```json
{
  "template_id": "uuid",              // обязательно
  "project_name": "Мой проект",       // опционально; если пусто — берётся имя шаблона
  "idempotency_key": "клиентский-uuid" // опционально, но РЕКОМЕНДУЕТСЯ (см. ниже)
}
```

### Про `idempotency_key` (защита от двойного списания)
- Это **ваш** уникальный ключ на одну логическую попытку «использовать шаблон».
- Если запрос оборвался по сети и вы **повторяете его с тем же `idempotency_key`** — сервер **не спишет
  деньги повторно**, а вернёт результат исходного списания.
- Если ключ не передать — **каждый** вызов будет считаться новой покупкой и спишет деньги заново.
- **Рекомендация:** генерируйте один UUID при нажатии кнопки «Использовать» и переиспользуйте его на все
  ретраи этого нажатия. На новое нажатие — новый ключ.

### Что происходит на сервере
1. Загружается шаблон, проверяется его валидность.
2. Если `price > 0` → **атомарно списывается** цена с баланса **текущего проекта** (конвертация в UZS по
   курсу валюты шаблона). Нехватка средств → `402`, проект **не создаётся**.
3. Создаётся новый проект + окружение + копируются данные/меню/страницы + публикуется микрофронтенд.
4. Если любой шаг после списания упал → деньги **автоматически возвращаются** (компенсация), приходит `500`.

### Успех — 200
```json
{
  "status": "OK",
  "data": {
    "project_id": "uuid",
    "ucode_project_id": "uuid",
    "environment_id": "uuid",
    "mcp_project_id": "uuid",
    "chat_id": "uuid",
    "api_key": "app-id",
    "resource_env_id": "uuid",
    "main_resource_env_id": "uuid",
    "microfrontend_id": "uuid",
    "microfrontend_repo_id": "uuid",
    "microfrontend_url": "https://...",
    "microfrontend_branch": "main",
    "template_preview_url": "https://...",
    "source_mcp_project_id": "uuid",
    "source_resource_env_id": "uuid",
    "source_mcp_resource_env_id": "uuid",
    "source_repo_id": "uuid",
    "source_function_id": "uuid",

    "charged_amount": 50000,             // ТОЛЬКО если шаблон был платным (в UZS)
    "charge_transaction_id": "uuid"      // ТОЛЬКО если было списание
  }
}
```
- `charged_amount` и `charge_transaction_id` присутствуют **только для платных шаблонов**. Для бесплатных
  их не будет — это нормально.

### Ошибки
| HTTP | status | Что показать пользователю |
|------|--------|---------------------------|
| 402 | `PAYMENT_REQUIRED` | **«Недостаточно средств на балансе»**. Предложить пополнить баланс. Проект НЕ создан, деньги НЕ списаны. |
| 404 | `NOT_FOUND` | Шаблон или проект не найдены. |
| 400 | `BAD_REQUEST` / `INVALID_ARGUMENT` | Невалидное тело / отсутствует `template_id`. |
| 401 | `UNAUTHORIZED` | Требуется вход. |
| 500 | `GRPC_ERROR` | Внутренняя ошибка. Если было списание — деньги уже возвращены автоматически; можно повторить. |

### Рекомендованный UX
1. На карточке платного шаблона показывать цену и (опционально) текущий баланс проекта.
2. По кнопке «Использовать» — генерировать `idempotency_key`, слать запрос, показывать лоадер (операция
   долгая — создаётся проект, копируются данные).
3. `402` → модалка «Недостаточно средств» + кнопка пополнения.
4. `200` → переход в созданный проект (`project_id` / `environment_id`).
5. При ретрае из-за таймаута — **тот же** `idempotency_key`.

## 9. Краткая шпаргалка эндпоинтов (фронт)

| Метод | Путь | Назначение | Авторизация |
|-------|------|-----------|-------------|
| GET | `/v1/ugen-template/public` | Каталог (публичный) | нет |
| GET | `/v1/ugen-template/public/:id` | Один шаблон (публичный) | нет |
| GET | `/v1/ugen-template` | Каталог (с реакцией юзера) | да |
| GET | `/v1/ugen-template/:id` | Один шаблон (с реакцией) | да |
| POST | `/v1/ugen-template` | Создать шаблон (без цены) | да |
| PUT | `/v1/ugen-template/:id` | Обновить шаблон (без цены) | да |
| POST | `/v1/ugen-template/:id/reaction` | Лайк/дизлайк | да |
| DELETE | `/v1/ugen-template/:id/reaction` | Снять реакцию | да |
| GET | `/v1/ugen-template/:id/reactions` | Список реакций | да |
| POST | `/v1/ugen-template/create-project` | **Использовать шаблон (+оплата)** | да |

> Установка цены (`PATCH /v1/ugen-template/:id/price`) — только в админ-панели, см. отдельную доку.
