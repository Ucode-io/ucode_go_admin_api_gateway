# Ugen Templates — АДМИН-ПАНЕЛЬ: цена за пользователя (per-user pricing)

Эта дока — про **вторую монетизацию** шаблона: **цену за каждого добавленного пользователя** в проект,
созданный из этого шаблона. Это дополнение к обычной цене шаблона (разовая цена импорта), которая описана
в `ugen_template_pricing_admin.md`.

> Обе цены задаются **одним и тем же эндпоинтом** `PATCH /v1/ugen-template/:id/price`. Эта дока фокусируется
> на новом поле `per_user_price`. Управляет ценами **только администратор**.

---

## 1. Суть — двухслойная монетизация шаблона

У шаблона теперь **две независимые** цены:

| Поле | Что это | Когда списывается | С кого |
|------|---------|-------------------|--------|
| `price` | Разовая цена **импорта** шаблона | 1 раз, при «использовать шаблон» (создание проекта) | С баланса head-проекта пользователя |
| `per_user_price` | Цена **за каждого пользователя** | Каждый раз при добавлении юзера в проект, созданный из шаблона | С баланса head-проекта |

Обе цены — в **одной валюте** шаблона (`currency_id`). Любую можно ставить `0`:
- `price = 0`, `per_user_price = 0` — полностью бесплатный шаблон.
- `price = 0`, `per_user_price = 2$` — импорт бесплатный, но каждый добавленный юзер стоит 2$.
- `price = 50 000`, `per_user_price = 0` — платный импорт, юзеры бесплатны (лимит по тарифу).
- `price = 50 000`, `per_user_price = 5 000` — платный импорт **и** платные юзеры.

### Как это работает дальше (сервер делает сам)
1. Админ задаёт `per_user_price` на **шаблоне**.
2. Когда пользователь импортирует шаблон, `per_user_price` **копируется в созданный проект** (вместе с валютой).
3. Когда в этот проект добавляют юзера: если `per_user_price > 0` — **тарифный лимит юзеров игнорируется**, а
   за каждого нового юзера списывается `per_user_price` с баланса head-проекта. Если `= 0` — всё как раньше
   (лимит по тарифу).

> ⚠️ **Важно:** `per_user_price` действует **только для проектов, созданных из шаблона**. У обычных проектов
> цены за юзера нет — они живут по тарифному лимиту. Отдельного «поставить цену на проект» **нет** — цена
> всегда наследуется из шаблона в момент импорта.

> 🎁 **Первый пользователь проекта — бесплатный.** При «использовать шаблон» списывается только разовая цена
> импорта (`price`); первый (owner) пользователь, создаваемый сразу после импорта, **не** тарифицируется.
> Плата `per_user_price` начинается со **второго** пользователя. То есть импорт шаблона `price=10$`,
> `per_user_price=2$` спишет ровно **10$**, а не 12$.

## 2. Общие правила

### Формат ответа (обёртка)
```json
{
  "status": "OK",
  "description": "The request has succeeded",
  "data": { ... },
  "custom_message": ""
}
```

### Авторизация
Стандартные заголовки админ-панели (как для остальных `/v1` admin-запросов):
```
Authorization: Bearer <admin_token>
Environment-Id: <environment_id>
Resource-Id: <resource_id>
```

### HTTP-коды
| Код | status | Когда |
|-----|--------|-------|
| 200 | `OK` | Успех |
| 400 | `BAD_REQUEST` / `INVALID_ARGUMENT` | Невалидный `:id`, кривой JSON, **отрицательная** `price` или `per_user_price` |
| 401 | `UNAUTHORIZED` | Нет/невалидный токен |
| 404/500 | `NOT_FOUND` / `GRPC_ERROR` | Шаблон не найден / внутренняя ошибка |

## 3. 💲 Установка / изменение цен шаблона

```
PATCH /v1/ugen-template/:id/price
```

### Path-параметр
- `:id` — UUID шаблона (обязательно, валидируется).

### Тело запроса
```json
{
  "price": 50000,
  "per_user_price": 5000,
  "currency_id": "88c816a3-24e8-4994-ab70-9bc826bb9dc3"
}
```

| Поле | Тип | Обязательность | Описание |
|------|-----|----------------|----------|
| `price` | number | да | Разовая цена импорта. **`>= 0`**. `0` = импорт бесплатный. |
| `per_user_price` | number | нет (по умолчанию 0) | Цена за каждого добавленного юзера. **`>= 0`**. `0` = юзеры бесплатны (лимит по тарифу). |
| `currency_id` | string (uuid) | нет | Валюта **обеих** цен. Пусто/не передано = **UZS**. |

> ⚠️ `price < 0` **или** `per_user_price < 0` → `400 BAD_REQUEST` («price cannot be negative»).
> ⚠️ Это операция типа **update/PATCH**: она перезаписывает `price`, `per_user_price` и `currency_id`
> значениями из тела. Если хотите поменять только одно — **всё равно передайте оба** актуальных значения
> (иначе не переданное поле уйдёт в 0). Рекомендуется всегда слать полный набор `{price, per_user_price, currency_id}`.

### Успех — 200
`data` = полный объект `UgenTemplate` с новыми ценами:
```json
{
  "status": "OK",
  "data": {
    "id": "9b1f...uuid",
    "name": "CRM Starter",
    "price": 50000,
    "per_user_price": 5000,
    "currency_id": "88c816a3-24e8-4994-ab70-9bc826bb9dc3",
    "...": "остальные поля шаблона"
  }
}
```

### Примеры

**Импорт бесплатный, юзеры по 2$ (валюта USD):**
```json
PATCH /v1/ugen-template/9b1f.../price
{ "price": 0, "per_user_price": 2, "currency_id": "<usd-uuid>" }
```

**Платный импорт и платные юзеры (UZS по умолчанию):**
```json
PATCH /v1/ugen-template/9b1f.../price
{ "price": 50000, "per_user_price": 5000 }
```

**Убрать плату за юзеров, импорт оставить платным:**
```json
PATCH /v1/ugen-template/9b1f.../price
{ "price": 50000, "per_user_price": 0 }
```

**Сделать шаблон полностью бесплатным:**
```json
PATCH /v1/ugen-template/9b1f.../price
{ "price": 0, "per_user_price": 0 }
```

## 4. Рекомендованный UX админки

Одна форма «Монетизация шаблона» с двумя полями:

```
┌─ Монетизация шаблона «CRM Starter» ───────────────┐
│ Валюта: [ UZS ▼ ]                                 │
│                                                   │
│ 💰 Цена импорта (разово)      [   50 000 ]        │
│    Списывается 1 раз при создании проекта.        │
│                                                   │
│ 👤 Цена за пользователя        [    5 000 ]       │
│    Списывается за каждого добавленного юзера       │
│    в проектах из этого шаблона.                   │
│                                                   │
│           [ Сохранить ]                           │
└───────────────────────────────────────────────────┘
```

Правила для UI:
1. Оба поля — числа `>= 0`. Валидируйте на клиенте до отправки (не пускайте отрицательные).
2. Валюта одна на оба поля — один селектор.
3. Всегда отправляйте полный набор `{price, per_user_price, currency_id}` (см. предупреждение в §3).
4. Показывайте текущие значения из объекта шаблона (`price`, `per_user_price`, `currency_id`) при открытии формы.
5. Подсказки-хинты под каждым полем: «0 = бесплатно / без платы за юзера (действует тарифный лимит)».

---

## 5. Что важно помнить админке

1. **Один эндпоинт** на всю монетизацию: `PATCH /v1/ugen-template/:id/price`.
2. `per_user_price` — цена **за юзера**, наследуется проектом **в момент импорта**. Изменение цены на шаблоне
   **не** меняет уже созданные проекты — они держат ту цену, что была на момент их создания.
3. Списание за юзеров идёт с баланса **head-проекта** пользователя; сервер сам конвертирует в UZS по курсу,
   списывает атомарно, и при сбое добавления юзера — возвращает деньги. Админке ничего делать не нужно.
4. Нет средств у пользователя → он получит `402` при добавлении юзера (см. фронтовую доку).
