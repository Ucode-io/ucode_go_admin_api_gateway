# Ugen Templates — документация для АДМИН-ПАНЕЛИ (управление ценой)

Эта дока описывает то, что делает **только администратор**: назначение и изменение **цены** шаблона.
Создание шаблонов и их использование пользователями — в отдельной доке `ugen_template_pricing_frontend.md`.

---

## 1. Суть

- Пользователи создают шаблоны **бесплатными** (`price = 0`) и **не могут** задать цену.
- **Только админ** назначает/меняет цену шаблона — отдельным эндпоинтом (по аналогии с админским
  управлением token packs).
- Установка цены — это **отдельная операция типа «update»**, а не часть создания шаблона.
- `price = 0` означает «сделать бесплатным снова».
- Когда пользователь позже создаёт проект из платного шаблона — цена списывается с баланса его проекта
  (эта логика на стороне сервера, админке ничего дополнительно делать не нужно).

---

## 2. Общие правила

### Формат ответа (обёртка)
```json
{
  "status": "OK",
  "description": "The request has succeeded",
  "data": { ... },
  "custom_message": ""
}
```

### Авторизация
Стандартные заголовки админ-панели (как для остальных `/v1` admin-запросов):
```
Authorization: Bearer <admin_token>
Environment-Id: <environment_id>
Resource-Id: <resource_id>
```

### HTTP-коды
| Код | status | Когда |
|-----|--------|-------|
| 200 | `OK` | Успех |
| 400 | `BAD_REQUEST` / `INVALID_ARGUMENT` | Невалидный `id` или тело; отрицательная цена |
| 401 | `UNAUTHORIZED` | Нет/невалидный токен |
| 404/500 | `NOT_FOUND` / `GRPC_ERROR` | Шаблон не найден / внутренняя ошибка |

## 3. 💲 Установка / изменение цены шаблона

```
PATCH /v1/ugen-template/:id/price
```

### Path-параметр
- `:id` — UUID шаблона (обязательно, валидируется).

### Тело запроса
```json
{
  "price": 50000,
  "currency_id": "88c816a3-24e8-4994-ab70-9bc826bb9dc3"
}
```
| Поле | Тип | Обязательность | Описание |
|------|-----|----------------|----------|
| `price` | number | да | Цена шаблона. **Должна быть `>= 0`**. `0` = сделать бесплатным. |
| `currency_id` | string (uuid) | нет | Валюта цены (uuid из справочника валют). Пусто/не передано = **UZS**. |

> ⚠️ `price < 0` → `400 BAD_REQUEST` («price cannot be negative»).

### Успех — 200
`data` = полный объект `UgenTemplate` с новыми `price` / `currency_id`:
```json
{
  "status": "OK",
  "data": {
    "id": "uuid",
    "name": "CRM Starter",
    "price": 50000,
    "currency_id": "88c816a3-24e8-4994-ab70-9bc826bb9dc3",
    "...": "остальные поля шаблона"
  }
}
```

### Ошибки
| HTTP | status | Причина |
|------|--------|---------|
| 400 | `INVALID_ARGUMENT` | Невалидный `:id` (не UUID) |
| 400 | `BAD_REQUEST` | Кривой JSON или `price < 0` |
| 404/500 | `NOT_FOUND` / `GRPC_ERROR` | Шаблон не найден или удалён / внутренняя ошибка |

### Примеры

**Назначить цену 50 000 UZS (валюта по умолчанию):**
```json
PATCH /v1/ugen-template/9b1f.../price
{ "price": 50000 }
```

**Назначить цену в другой валюте:**
```json
PATCH /v1/ugen-template/9b1f.../price
{ "price": 5, "currency_id": "88c816a3-24e8-4994-ab70-9bc826bb9dc3" }
```

**Сделать шаблон снова бесплатным:**
```json
PATCH /v1/ugen-template/9b1f.../price
{ "price": 0 }
```

## 4. Что важно помнить админке

1. **Только этот эндпоинт** управляет ценой. `POST`/`PUT` шаблона цену не меняют (полей цены там нет).
2. Валюту (`currency_id`) берите из справочника валют платформы. Если оставить пустым — цена в UZS.
3. При создании проекта из платного шаблона сервер сам:
   - конвертирует цену в UZS по курсу;
   - атомарно списывает с баланса проекта пользователя (нехватка → `402` пользователю);
   - при сбое создания — автоматически возвращает деньги.
   Админке для этого ничего делать не нужно.
4. Изменение цены влияет только на **будущие** создания проектов; уже созданные проекты не затрагиваются.
